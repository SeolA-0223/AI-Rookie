# 웹 빠른 시작

## 누구를 위한 문서인가
- 개발 환경을 이미 켜 둔 상태에서 대시보드를 어떻게 쓰는지 빠르게 확인하려는 사용자
- 샘플 체험과 실제 조례 조회의 차이를 한 번에 이해하려는 사용자

## 가장 쉬운 시작
1. 서버를 실행합니다.
   - `npm run start`
2. 브라우저에서 `http://127.0.0.1:3000`을 엽니다.
3. 기본 `샘플 체험` 모드에서 케이스 팩을 하나 고릅니다.
4. `샘플 케이스 분석하기`를 누릅니다.
5. 아래 결과 패널을 `변경 요약 -> 영향 문서 -> 위험 우선순위 -> 초안 미리보기` 순서로 확인합니다.

## 샘플 체험 모드
- 번들 케이스 팩만 사용합니다.
- 조례 ID를 알 필요가 없습니다.
- 데모, 발표, 구조 이해용으로 가장 적합합니다.
- 현재 포함된 케이스는 울산, 부천, 서울, 대전 시나리오입니다.

## 실제 조례 조회 모드
1. 상단 모드를 `실제 조례 조회`로 바꿉니다.
2. provider를 고릅니다.
   - `law.go.kr 공개`: 공식 공개 검색 경로를 사용할 때
   - `Korea Law MCP`: 별도 MCP endpoint가 준비되어 있을 때
3. 조례명을 검색합니다.
4. 추천 조합이 보이면 `추천 조합 적용`을 눌러 before/after ID를 자동 입력합니다.
5. 추천이 없으면 검색 결과 카드에서 `이전 버전으로 사용`, `개정 버전으로 사용`을 눌러 직접 채웁니다.
6. 두 ID가 채워졌으면 `선택한 조례 분석하기`를 누릅니다.

## 고급 설정과 출처 확인
- 기본 사용 흐름을 방해하지 않도록 접힌 영역으로 이동했습니다.
- 여기서 확인할 수 있는 항목:
  - provider 연결 상태
  - live probe 결과
  - search provenance
  - 최근 분석 입력 source

## 결과 읽는 방법
- `변경 요약`: 조항별 변경 내용을 먼저 파악합니다.
- `영향 문서`: 어떤 내부 문서가 같이 수정되어야 하는지 봅니다.
- `위험 우선순위`: 잘못 안내될 위험이 큰 항목부터 정렬합니다.
- `초안 미리보기`: 내부 공지, 시민 안내문, FAQ, 비교표를 검토합니다.

## 자주 쓰는 상황
- 빠른 데모: `샘플 체험`에서 바로 실행
- 실제 법령 비교: `실제 조례 조회`에서 검색 후 추천 pair 적용
- 근거 확인: `고급 설정과 출처 확인` 열기
- 이전 결과 재사용: `최근 실행 이력 보기` 열기

## 주의
- 현재 분석 결과 본문은 소스 문구 언어를 유지합니다.
- `law.go.kr 공개` 검색은 demo `LAW_GO_OC=test` 환경에서 결과가 비거나 부정확할 수 있습니다.
- 실사용 검색 품질이 필요하면 실제 `LAW_GO_OC` 또는 정확한 조례 ID를 사용하는 편이 안전합니다.
